# 天翼视联助手 Chrome 插件

一款专为天翼视联平台（vcp.21cn.com）用户设计的 Chrome 浏览器插件，提供批量级联、设备查询、画质监测等实用功能，大幅提升设备管理效率。

## 功能特性

### 1. 用户信息（侧边栏首页）
- 自动读取当前 VCP 页面登录信息
- 显示用户 ID（userId）、手机号（account）
- 一键刷新当前页面数据

### 2. 批量级联
- 支持手动输入设备编码（每行一个）
- 支持上传 CSV 文件批量导入
- 提供 CSV 模板下载
- 批量发送级联请求（batchSize = 80）
- 实时显示处理进度
- 导出级联结果清单（CSV）
- 支持中途停止操作

### 3. 设备在线状态查询
- 支持手动输入设备编码
- 支持上传 CSV 文件批量导入
- 批量查询设备在线状态
- 导出查询结果（CSV）

### 4. 设备清单查询
- 输入手机号获取企业主列表
- 选择企业主后分页查询全部设备
- 表格展示设备信息（序号、设备名称、设备编码、设备型号、目录）
- 支持导出设备清单（CSV）
- 显示查询进度，支持中途停止

### 5. 画质监测
- 输入设备编码查询设备截图
- 点击图片放大查看
- 支持关闭放大窗口（点击 × 按钮或背景区域）

### 6. 监控目录调整
- 查询客户列表和区域目录树
- 支持动态加载子目录
- 选择目标目录后执行调整操作

### 7. 批量级联-2（汇聚功能）⭐ 新功能
- **支持多目录级联**：同一手机号下的设备可级联到不同目标目录
- **智能数据预处理**：按手机号+目标目录自动二次分组
- **CSV模板下载**：提供标准格式模板（设备编码、手机号、目标目录）
- **完整异常处理**：确保每台设备都有处理记录（企业主不存在、路数不足等）
- **5步级联流程**：
  1. 检查企业主级联剩余路数
  2. 自动创建/确保目标目录存在
  3. 获取用户regionCode（用于设备查询）
  4. 批量查询设备详细信息
  5. 将设备级联到指定目标目录
- **实时进度追踪**：按目录组为单位显示处理进度
- **详细结果展示**：包含手机号、目标目录、设备码、状态、原因
- **支持结果导出**：生成完整的CSV处理报告

## 安装方法

1. 打开 Chrome，访问 `chrome://extensions/`
2. 开启右上角 **开发者模式**
3. 点击 **加载已解压的扩展程序**
4. 选择本插件所在目录
5. 点击浏览器工具栏右侧的插件图标，或通过**边栏**打开使用

## 使用前提

- 仅适用于 **https://vcp.21cn.com/** 域名下的页面
- 使用前需先在 VCP 平台登录账号

## 批量级联-2（汇聚功能）使用指南

### 功能简介
批量级联-2是专为复杂级联场景设计的增强功能，支持将同一企业主的设备级联到**不同的目标目录**，适用于多项目、多场景的设备管理需求。

### CSV文件格式

```csv
设备编码,汇聚手机号码,目标目录
511102004413240001,13800138000,测试目录
511102004413240002,13800138000,测试目录
511102004413240003,13800138000,生产目录
511102004413240004,13900139000,研发目录
```

### 数据预处理逻辑
系统会自动按**手机号+目标目录**进行二次分组：

```
📱 13800138000 (7台设备, 2个目录)
├── 📁 测试目录 (5台)
│   └── 设备: 5111020044..., 5111020044..., ...
└── 📁 生产目录 (2台)
    └── 设备: 5111020044..., 5111020044...
```

### 处理流程（每个目录组）

1. **检查级联容量** - 验证企业主剩余路数是否足够
2. **确保目录存在** - 自动创建目标目录（如不存在）
3. **获取regionCode** - 查询用户区域编码（用于设备查询）
4. **查询设备信息** - 批量获取设备详细信息
5. **执行级联操作** - 将设备级联到指定目标目录

### 异常处理机制
- ✅ 企业主不存在 → 所有设备记录失败原因
- ✅ 级联路数不足 → 标记并跳过该目录组
- ✅ 设备查询失败 → 单独标记"设备不存在"
- ✅ 单台级联失败 → 不影响其他设备

### 结果展示
处理完成后显示包含以下信息的表格：
- 手机号码、目标目录、设备编码、级联状态、状态信息
- 支持导出为CSV文件用于后续分析

## 项目结构

```
tysl/
├── manifest.json       # 插件配置文件
├── sidepanel.html     # 侧边栏主页面
├── popup.js           # 侧边栏业务逻辑（含所有功能模块）
├── content.js         # 内容脚本（注入 VCP 页面，执行 API 请求）
├── background.js      # 后台服务脚本
├── floating-window.js # 浮窗功能（预留）
└── icons/             # 插件图标
```

## 核心模块说明

### popup.js 主要函数

| 函数 | 功能 |
|------|------|
| `fetchData()` | 获取当前页面用户信息 |
| `autoFillUserId()` | 自动填充用户 ID |
| `getCustomList()` | 获取客户列表（企业主筛选） |
| `getUserRegionList()` | 获取用户区域代码 |
| `getLevelCusRegion()` | 获取监控目录树 |

| `queryDeviceStatus()` | 查询设备在线状态 |

| `ensureVcpTab()` | 统一 VCP 标签页验证与脚本注入 |
| `sendVcpMessage()` | 统一消息发送到 content.js |

#### 汇聚功能2（批量级联-2）核心函数

| 函数 | 功能 |
|------|------|
| `groupByPhoneAndDir()` | 按手机号+目标目录进行二次分组 |
| `executeCascade2ForDirGroup()` | 对单个目录组执行5步级联流程 |
| `executeCascade2ForPhone()` | 对单个手机号的所有目录组执行级联 |
| `startCascade2()` | 汇聚功能2主入口，协调整个处理流程 |
| `readCascade2CSVFile()` | 读取并解析汇聚功能2的CSV文件 |
| `processCascade2CSVFile()` | 处理上传文件并显示分组预览 |
| `ensureRegionExists()` | 确保目标目录存在（不存在则创建） |
| `getCustomListForCascade2()` | 查询企业主信息（用于级联） |
| `getLevelCusRegionForCascade2()` | 查询企业主监控目录（用于创建目标目录） |
| `getDeviceInfoForCascade2()` | 批量查询设备详细信息 |
| `displayCascade2Results()` | 显示汇聚功能2的处理结果表格 |
| `exportCascade2Result()` | 导出汇聚功能2的结果为CSV |
